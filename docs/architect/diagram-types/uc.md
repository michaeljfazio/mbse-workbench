- Use case shape (ellipse); actor shape (stick figure for human, alternative shapes for non-human actors).
- Association between actor and use case (solid line, optionally with arrow showing initiation direction).
- `<<include>>` relationship (dashed arrow from including use case to included).
- `<<extend>>` relationship (dashed arrow from extending use case to base, with extension point).
- Generalization between use cases (solid line with hollow triangle arrowhead).
- Generalization between actors (same hollow-triangle notation).
- System boundary rectangle with system name in top-left.
- Common non-standard but widely-adopted conventions (Cameo / Capella / MagicDraw / Papyrus).
- Common pitfalls and reader-expectation traps.
